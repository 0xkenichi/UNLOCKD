# Controlled Mainnet Rollout Plan

Last updated: 2026-02-13
Owner: Founder office

This plan defines the phased path from testnet pilots to paid mainnet cohorts.

## Rollout Principles

- No unrestricted rollout before audit closeout.
- Start with small cohort and conservative limits.
- Measure weekly; expand only when guardrails hold.

## Entry Criteria

- External audit completed and critical/high findings remediated.
- Incident runbook tested for on-call and emergency controls.
- Weekly KPI reporting stable for at least 4 consecutive weeks.

## Phases

Phase 0: Mainnet rehearsal
- Dry-run deployments and rollback rehearsal
- Partner onboarding simulation with production-like checks

Phase 1: Controlled cohort
- 2 paying pilot counterparties max
- Conservative collateral set and LTV profile
- Daily operations checks in first two weeks

Phase 2: Measured expansion
- Expand only if phase-1 KPIs remain within limits
- Add one channel partner at a time

## Guardrail KPIs

- Bad debt ratio <= 1%
- Incident severity 0 count = 0
- Repayment behavior at or above pilot baseline
- Time-to-resolution within runbook targets

## Rollback Triggers

- Any unresolved critical security issue
- Bad debt ratio above guardrail for two consecutive periods
- Repeated Sev 1 incidents from same root cause

## Monthly Reporting

- Treasury and fee routing summary
- Cohort-level risk and growth metrics
- Open risk register and remediation status
