# ASI:Create / ASI:One Integration

This doc describes how to add **ASI:Create** (and the developer-facing **ASI:One**) to the Vestra project.

## What are ASI:Create and ASI:One?

- **ASI:Create** is the [Artificial Superintelligence Alliance](https://superintelligence.io/) platform for funding, creating, deploying, and monetizing AI agents. It offers a multi-modal interface, decentralized compute, and agent discovery.
- **ASI:One** is [Fetch.ai](https://fetch.ai)’s LLM and developer platform that powers the ASI ecosystem. It exposes an OpenAI-compatible API and connects to domain-expert agents via [Agentverse](https://agentverse.ai/) and the uAgents chat protocol.

## Option 1: Use ASI:One as the in-app agent LLM (done in this repo)

The backend chat agent (`/api/agent/chat`) already supports ASI:One as an LLM backend. When configured, it is tried first before other providers.

### Setup

1. **Get an API key**
   - Sign up at [ASI:One Platform](https://asi1.ai/).
   - In the Developer section, create an API key and save it securely.

2. **Configure the backend**
   - In `backend/.env` (or your environment), set:
     - `ASI_ONE_API_KEY=<your-key>`
   - Optional:
     - `ASI_ONE_BASE_URL=https://api.asi1.ai/v1` (default)
     - `ASI_ONE_MODEL=asi1` (or `asi1-mini`, `asi1-fast`, `asi1-extended`, `asi1-agentic`, `asi1-graph`)

3. **Run the backend**
   - Start the server; the in-app agent will use ASI:One for chat completions when the key is set. The existing RAG (docs, intents, price behavior, tools) is unchanged; only the final reply is generated by ASI:One.

## Option 2: Publish Vestra as an ASI:One-compatible agent

To make the Vestra agent **discoverable in ASI:One Chat** and the broader ASI:Create ecosystem (and eventually monetizable), you can expose it as an agent that speaks the **chat protocol** and is registered on **Agentverse**.

### High-level steps

1. **Agentverse and mailbox**
   - Sign up at [Agentverse](https://agentverse.ai/) and create a [mailbox](https://uagents.fetch.ai/docs/agentverse/mailbox) for your agent so it can receive messages from ASI:One and other agents.

2. **Implement the chat protocol**
   - The ASI:One ecosystem expects the [uAgents Chat Protocol](https://uagents.fetch.ai/docs/examples/asi-1): handle `ChatMessage`, respond with `ChatAcknowledgement` and then a reply `ChatMessage` (e.g. with `TextContent` and optionally `EndSessionContent`).

3. **Two implementation approaches**
   - **A. Adapter service (recommended)**  
     - Keep the existing Node backend and RAG.  
     - Add a small **Python uAgents** service that:
       - Listens for chat protocol messages via the mailbox.
       - For each user message, calls your Vestra backend (e.g. `POST /api/agent/chat` or an internal function), then sends the reply back over the chat protocol.
     - Register this Python agent on Agentverse with the mailbox; it becomes the “Vestra expert” in ASI:One.
   - **B. Full uAgents agent**  
     - Reimplement or mirror the RAG + tools logic in Python inside a uAgents agent, and optionally call ASI:One’s API for the final answer (as in the [ASI:One chat example](https://uagents.fetch.ai/docs/examples/asi-1)).

4. **Install uAgents**
   - Follow [uAgents install](https://uagents.fetch.ai/docs/getting-started/install). The chat protocol is in `uagents_core.contrib.protocols.chat` (`ChatMessage`, `ChatAcknowledgement`, `TextContent`, etc.).

5. **Run and register**
   - Run your agent with `mailbox=True` and `publish_agent_details=True`.
   - Use the [Agent inspector](https://uagents.fetch.ai/docs/agentverse/inspector) to connect the mailbox and verify registration.
   - In Agentverse, set name/handle and profile so the agent is discoverable (e.g. “Vestra” / “@vestra” for vesting-backed borrowing and protocol Q&A).

6. **Discoverability**
   - See [Agents README Guidelines](https://docs.agentverse.ai/documentation/agent-discovery/readme-guidelines) and [Agent setup / ranking](https://docs.agentverse.ai/documentation/agent-discovery/agent-setup-guide) to improve visibility in the ASI:One / ASI:Create ecosystem.
